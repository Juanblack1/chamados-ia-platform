# ADR 001: Extract App State Screens

## Status

Accepted

## Context

The root app component should orchestrate session, data and route state. Boot, login and loading fallback screens are UI state components and do not need to live inline in `App.tsx`.

## Decision

Move app state screens to `frontend/src/components/AppStateScreens.tsx`.

## Consequences

- `App.tsx` becomes smaller and more focused.
- Login UI remains reusable and easier to change without touching app orchestration.
- No behavior change is intended.
