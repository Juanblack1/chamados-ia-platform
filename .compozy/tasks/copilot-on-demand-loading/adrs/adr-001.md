# ADR-001: Load optional Copilot UI only on demand

## Status

Accepted

## Context

The Copilot provider and popup are optional for the normal service-desk workflow, but mounting the provider around the authenticated app forces Copilot code to load with the workspace. The build output contains large Copilot-related chunks, including markdown, code highlighting and diagram support.

## Decision

Move Copilot provider and popup into a lazy component mounted only after the user clicks the Copilot button.

## Consequences

- The service-desk workspace is no longer gated by optional Copilot UI loading.
- Copilot remains available from the same topbar button.
- The implementation preserves the existing runtime URL, credentials, labels and attachment limits.
