# ADR-001: Show ticket-level AI traceability in the workspace

## Status

Accepted

## Context

`docs/specs/agent-governance.sdd.md` requires the UI to show model gateway route, trace ID, API scope, policy status, and human override affordance. Tickets already carry trace ids in decision metadata, and the frontend already loads trace spans, but the current treatment workspace does not expose this as a clear operational capsule next to the AI decision.

## Decision

Add a compact traceability panel inside the ticket workspace. The panel will reuse existing ticket metadata and loaded spans. Backend decision metadata will include model route and execution mode so the UI can distinguish external model cascade from deterministic fallback.

## Consequences

- Positive: analysts can verify why and how an AI decision was produced without leaving the ticket.
- Positive: governance acceptance criteria become visible in the main workflow.
- Positive: no new persistence subsystem is required.
- Tradeoff: this cycle shows current trace data only; historical trace search remains outside scope.
