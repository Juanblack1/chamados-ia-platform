# ADR-001: Align RAG catalog with evidence source IDs

## Status

Accepted

## Context

The workspace cites RAG evidence by source id, while the admin catalog exposes knowledge articles with different ids. This creates a governance gap: an operator can see that a decision used `kb-erp-billing-lock`, but the catalog lists `kb-erp-001`, so usage, freshness, and ownership cannot be reconciled.

The source PDF at `C:\Users\veron\Downloads\arquitetura_de_solucoes_com_ia.pdf` was previously found to be scanned with no extractable text through available local tooling. This cycle uses the repository implementation, existing product docs, and earlier research on AI operational feedback loops as source context.

## Decision

Use the actual RAG source ids as catalog article ids and add lightweight governance metadata to each article:

- `ownerGroupId`
- `reviewCadenceDays`
- `status`

The governance screen will compute knowledge health from current tickets, retrieved sources, and the catalog. No new persistence subsystem is introduced in this cycle.

## Consequences

- Positive: cited evidence can be matched to catalog entries.
- Positive: governance can surface uncataloged sources, stale articles, and service gaps.
- Positive: scope remains compatible with memory and Redis ticket storage because catalog data is static.
- Tradeoff: article editing remains outside this cycle; the new UI is diagnostic, not CRUD.
