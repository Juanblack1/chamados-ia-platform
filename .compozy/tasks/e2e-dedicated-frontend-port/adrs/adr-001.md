# ADR-001: Prefer deterministic E2E frontend startup over server reuse

## Status

Accepted

## Context

Local Playwright server reuse accepted an unrelated app already listening on the generic Vite port. The tests then waited for Chamados-ia login fields on a different UI.

## Decision

Use a dedicated default frontend E2E port, start Vite with `--strictPort`, and set frontend `reuseExistingServer` to `false`.

## Consequences

- E2E targets the intended app.
- Port conflicts fail early with a clear startup problem.
- Repeated local runs rely on Playwright-managed process cleanup instead of arbitrary server reuse.
