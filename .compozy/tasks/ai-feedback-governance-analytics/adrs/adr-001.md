# ADR-001: Aggregate ticket-local AI feedback in governance

## Status

Accepted

## Date

2026-06-04

## Context

Service Desk IA now allows analysts to record structured feedback on AI triage and resolution draft decisions. That feedback is visible inside a ticket, but governance users cannot see aggregate quality signals across the queue. The dashboard already combines tickets, traces, RAG coverage, evals, and audit events, so feedback analytics belongs there.

Market and operations research reinforces the need to combine automated evals with human feedback. LangSmith describes human evaluation and annotation as part of continuous agent improvement. Datadog LLM Observability includes automated evaluations and human feedback in its evaluation workflow. OECD AI accountability guidance emphasizes reporting channels, human oversight, and continuous monitoring.

## Decision

Build the first feedback analytics layer in the frontend governance model using `ticket.ai.feedback` from already loaded tickets. The panel will summarize ratings, decision types, recent comments, and recommended follow-up when negative feedback appears.

## Alternatives Considered

### New backend aggregate endpoint

- Pros: centralizes aggregation and can later support persistence.
- Cons: unnecessary for a dataset already loaded by governance, adds API surface before the model needs it.
- Rejected for this cycle.

### Persisted feedback trend store

- Pros: supports historical trend charts.
- Cons: requires storage, retention, and migration design.
- Rejected because the product first needs current operational visibility.

### Keep feedback ticket-local

- Pros: no implementation work.
- Cons: managers cannot see decision quality patterns without opening each ticket.
- Rejected because it leaves the human feedback loop incomplete.

## Consequences

- Positive: governance can prioritize incorrect or review-needed AI decisions.
- Positive: no new backend persistence or data migration.
- Trade-off: analytics only covers the currently authorized ticket set.
- Risk: sparse feedback may appear as a weak metric. Mitigation: show sample size clearly.
