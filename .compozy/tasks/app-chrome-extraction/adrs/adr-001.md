# ADR-001: Keep app chrome in a dedicated component module

## Status

Accepted

## Context

The authenticated app shell has navigation and filtering controls that change independently from ticket/user business logic. Keeping these components inside `App.tsx` makes the orchestrator larger and increases the risk of touching unrelated flows when updating chrome.

## Decision

Move sidebar and topbar rendering to `frontend/src/components/AppChrome.tsx`, exporting the small shared types needed by `App.tsx`.

## Consequences

- Navigation and queue-filter UI have a clearer ownership boundary.
- `App.tsx` becomes smaller and more focused on state/view orchestration.
- Existing CSS and behavior stay unchanged.
