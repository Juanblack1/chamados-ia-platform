# ADR-001: Keep App focused on orchestration, not admin form implementation

## Status

Accepted

## Context

`App.tsx` still mixes app shell orchestration with detailed profile and user administration forms. The active user-management block is cohesive and belongs under `views/admin`.

## Decision

Move routed profile/user-management views to `frontend/src/views/admin/UserManagementViews.tsx` and export only the screen-level components needed by `App.tsx`.

## Consequences

- `App.tsx` becomes smaller and easier to reason about.
- Admin UI helpers stay colocated with the admin surface.
- Future user-management changes can happen without touching the app root.
