# ADR-001: Add A Client-Side Workload Panel To The Queue

## Status

Accepted

## Context

The queue already receives all tickets authorized for the current user and applies client-side filters for status, priority, SLA, confidence, and search. Managers need a fast read on where active workload is concentrated before they open individual tickets.

## Decision

Compute workload summaries in the queue view from `allTickets` and render a read-only workload panel. The panel will show group load and assignee load using active tickets only.

## Consequences

- Positive: No backend route is required for the MVP.
- Positive: The panel stays aligned with the current frontend filtering model.
- Positive: It improves operational scanning without introducing assignment mutation risk.
- Negative: It does not support historical trend analysis.
- Negative: Very large ticket volumes may eventually need a server-side analytics endpoint.
