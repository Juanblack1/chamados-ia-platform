# ADR-001: Remove inactive App-local admin surfaces before larger extraction

## Status

Accepted

## Context

The current routed user administration flow uses `UserDirectoryView`, `UserCreatePage` and `UserDetailPage`. Older `UsersView` and `AdminView` implementations remain in `App.tsx` without active routes.

## Decision

Delete inactive admin/user surfaces and duplicate local helpers before attempting broader user-management extraction.

## Consequences

- `App.tsx` becomes smaller and easier to extract safely later.
- Shared SLA badge behavior has one source.
- Active routes and behavior remain unchanged.
