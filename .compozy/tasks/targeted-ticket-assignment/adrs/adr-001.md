# ADR-001: Validate Targeted Assignment In The Orchestrator

## Status

Accepted

## Context

The ticket assignment endpoint parses `assigneeId` but ignores it. Adding target assignment only at the UI level would leave API behavior misleading and could allow invalid targets if validation were placed solely in the route.

## Decision

Resolve the target user in the route and enforce assignment compatibility inside `AgentOrchestrator.assignTicket`.

## Consequences

- Positive: API and UI behavior match.
- Positive: Assignment rules are enforced even if another client calls the endpoint.
- Positive: Self-assignment remains the default when no target is provided.
- Negative: Ticket routes now depend on the auth store.
