# ADR 001: Preserve `lib/api` as the public API module

## Status

Accepted

## Context

The frontend already imports API types from `lib/api` across route views and components. Moving every consumer to a new path would make a mechanical extraction look larger than the architectural change itself.

## Decision

Move type declarations to `apiTypes.ts`, but keep `api.ts` as the public import path by reexporting those types.

## Consequences

- Existing consumers keep working without churn.
- `api.ts` becomes focused on network behavior.
- Future modules can import from `apiTypes.ts` directly only when there is a clear reason.
